## PR 정책

1. 브랜치 이름: feature/[주요기능 이름], style/[스타일링 하는 컴포넌트], build/[빌드에 추가되는 것]
ex) feature/login, feature/signup, style/header, build/swiper

2. 해당 브랜치에서 main 브랜치로 Pull Request 생성 후에, 코드리뷰 요청

3. 확인 후, 머지
